Slicer

A library by FELD for the Processing programming environment.
Last update, 06/18/2013.

A library for slicing geometry, well, into slices.
Takes input geometry from model files or directly from things drawn in Processing and slices them. The result can be images or paths.
Feel free to replace this paragraph with a description of the library.
Contributed libraries are developed, documented, and maintained by members of the Processing community. Further directions are included with each library. For feedback and support, please post to the Discourse. We strongly encourage all libraries to be open source, but not all of them are.

Download

Download Slicer version 0.1b (1) in .zip format.

Installation

Unzip and put the extracted Slicer folder into the libraries folder of your Processing sketches. Reference and examples are included in the Slicer folder.

Keywords. slicing, geometry, pixel, path, vector

Reference. Have a look at the javadoc reference here. A copy of the reference is included in the .zip as well.
